HOW DATA FLOWS:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  1. User types query in Browser (port 8501)
                    ↓
  2. Streamlit sends to API Server (port 8000)
                    ↓
  3. API Server runs RAG chain:
     - Retrieves papers from ChromaDB
     - Sends to Mistral-7B for code generation
                    ↓
  4. Generated code returns to Streamlit
                    ↓
  5. Browser displays PyTorch code


STEP-BY-STEP ACTION PLAN:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

[ ] STEP 1: Open a terminal
    
    In VS Code: Terminal → New Terminal
    In Windows: Right-click folder → Open in Terminal

[ ] STEP 2: Run this command:
    
    uvicorn src.server:app --reload
    
    ⏳ Wait for: "Application startup complete"
    ⚠️  KEEP THIS TERMINAL OPEN!

[ ] STEP 3: Open ANOTHER terminal (keep first one running!)
    
    In VS Code: Click the + button in terminal panel
    In Windows: Open another PowerShell window

[ ] STEP 4: In the NEW terminal, run:
    
    streamlit run src/frontend.py
    
    ✓ Browser opens automatically
    ⚠️  KEEP THIS TERMINAL OPEN TOO!

[ ] STEP 5: In browser, try a query:
    
    Example: "implement transformer attention mechanism"
    Click "Generate Code"
    Wait 30-60 seconds


COMMON MISTAKES TO AVOID:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

❌ Only starting Streamlit (Terminal 2) but not API server (Terminal 1)
   → You get connection errors!

❌ Closing Terminal 1 after starting it
   → API server stops, connection breaks

❌ Running both commands in the SAME terminal
   → Second command never runs because first one blocks

❌ Forgetting to wait for "Application startup complete"
   → Server isn't ready yet, requests fail
